# IClude 对外产品路线图

## 1. 文档目的

这份文档面向以下读者：

- 日常用户
- 开发者
- 潜在合作方
- 想快速理解 IClude 当前状态的人

它只回答 4 个问题：

1. IClude 现在是什么
2. IClude 现在能做什么
3. 接下来会优先把什么做好
4. 企业能力会不会做，以及什么时候做

---

## 2. 当前定位

IClude 是一个开源、本地优先的记忆系统，服务对象首先是日常用户和开发者。

核心价值不是“替模型回答一切”，而是让 AI 应用：

- 记得住长期信息
- 在需要时找得到
- 能基于历史记忆做更稳定的反思和检索

当前版本已经不是概念验证，而是可交付版本。

**当前结论：Phase 2 已完成，IClude 已可提供给日常用户使用。**

---

## 3. 当前已具备的能力

### 3.1 记忆写入

- 存储结构化记忆内容和元数据
- 支持 SQLite / Qdrant / 混合存储
- 支持对话摄取和文档入库
- 支持记忆生命周期、强化、软删除与恢复

### 3.2 记忆召回

- 支持全文检索
- 支持向量检索
- 支持图谱关联检索
- 支持三路混合融合
- 支持 token 裁剪和细节等级控制

### 3.3 记忆理解

- 支持 Reflect 反思式检索
- 支持自动实体抽取
- 支持时间、关系、上下文等多种检索信号

### 3.4 开放接入

- 提供 REST API
- 提供 Python SDK
- 提供 MCP Server，便于 AI 助手接入

---

## 4. 接下来 2 条路线

为了避免“用户能力”和“企业能力”互相干扰，后续路线拆成两条独立轨道。

### 4.1 Benchmark Track

这是开源主线，也是日常用户最直接受益的一条路线。

目标：

- 提升记忆召回质量
- 提升复杂问题下的反思能力
- 在 LongMemEval 上持续逼近行业公开高分

当前技术标杆：

- `Hindsight 91.4%`

但这里的含义是：

- 它是长期技术标杆
- 不是对外承诺
- 也不是短期版本发布时间表

对用户来说，这条路线带来的实际收益是：

- 多会话问题答得更稳
- 时间相关问题答得更准
- 复杂检索结果更少“明明记住了却没用上”

### 4.2 Enterprise Track

这是企业版路线，不阻塞日常用户版本。

主要覆盖：

- 认证与权限
- 管理后台
- 监控与运维
- 多租户隔离
- 企业交付能力

这条路线存在，但不会影响开源主线继续提升记忆质量。

---

## 5. 接下来会优先做什么

### 第一优先级：把评测做扎实

- 建立可复现的 LongMemEval 评测闭环
- 固化基线，保证每次优化都能量化比较

### 第二优先级：把检索精排做强

- 引入 cross-encoder 重排
- 优化时间敏感查询
- 提高复杂问题下的候选质量

### 第三优先级：把 Reflect 的“利用记忆能力”做强

- 让多轮反思能保留前一轮 reasoning
- 减少“召回到了但没用好”的情况

### 第四优先级：逐步补齐企业能力

- 不阻塞用户版节奏
- 按企业价值独立排期

---

## 6. 对外口径

对外推荐统一这样描述：

> IClude 当前已经完成日常用户可用版本。后续将沿两条路线推进：一条是持续提升记忆质量、对标 Hindsight 91.4% 的 Benchmark Track；另一条是独立推进认证、后台、监控和多租户的 Enterprise Track。

---

## 7. 当前不建议对外承诺的内容

以下内容可以作为方向讲，但不建议作为近期对外承诺：

- 短期内超过 `Hindsight 91.4%`
- 完整企业权限体系已就绪
- 完整企业管理后台已可用
- 全量多租户和合规能力已经完成

更稳妥的说法应该是：

- 已有可用版本
- 正在用可复现 benchmark 持续优化
- 企业能力按独立路线推进

---

## 8. 一句话总结

IClude 现在应该被表述为：**一个已经可供日常用户使用、并以 Hindsight 91.4% 为长期技术标杆持续优化的本地优先记忆系统。**
