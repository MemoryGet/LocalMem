# IClude 安装与使用指南

> IClude 是一个 local-first 的 AI 记忆系统，为 Claude Code 提供跨会话的持久记忆能力。

## 快速安装

### macOS / Linux

```bash
curl -fsSL https://raw.githubusercontent.com/MemoryGet/LocalMem/main/install.sh | bash
```

### Windows (PowerShell)

```powershell
irm https://raw.githubusercontent.com/MemoryGet/LocalMem/main/install.ps1 | iex
```

### 安装脚本自动完成的事项

1. 检测平台和 CPU 架构（linux/darwin/windows × amd64/arm64）
2. 从 GitHub Releases 下载 `iclude-mcp` 和 `iclude-cli` 二进制文件
3. 安装到 `~/.iclude/bin/` 并加入 PATH
4. 交互式生成 `~/.iclude/config.yaml`（提示输入 OpenAI API Key，可跳过）
5. 配置 Claude Code MCP（stdio 模式，写入 `~/.claude/.mcp.json`）
6. **重启 Claude Code 即生效**

### 安装后目录结构

```
~/.iclude/
  bin/
    iclude-mcp      # MCP Server 二进制
    iclude-cli      # CLI Hooks 二进制
  config.yaml       # 配置文件
  iclude.db         # SQLite 数据库（首次运行时自动创建）

~/.claude/
  .mcp.json         # Claude Code MCP 配置（自动写入）
```

## 工作原理

IClude 通过 MCP (Model Context Protocol) 与 Claude Code 集成，提供三个自动化 hook：

```
Claude Code 会话开始
   │
   ├─ SessionStart hook → 注入最近 20 条记忆摘要到对话上下文
   │
   ├─ PostToolUse hook  → 静默捕获每次工具调用，存为结构化记忆
   │                       （自动去重、摘要生成、实体抽取）
   │
   └─ Stop hook         → 生成会话总结并保存
```

同时，Claude Code 可以在对话中直接使用 MCP 工具进行记忆操作。

## MCP 工具一览

| 工具 | 用途 | 使用场景 |
|------|------|---------|
| `iclude_scan` | **首选搜索工具**。返回紧凑索引（ID、标题、分数、标签） | 搜索记忆时首先使用，节省 ~10x token |
| `iclude_fetch` | 按 ID 获取完整记忆内容（最多 20 条） | 在 scan 后获取感兴趣的条目详情 |
| `iclude_retain` | 保存新记忆 | 保存重要信息、笔记、观察 |
| `iclude_recall` | 全文+语义搜索，返回完整内容 | 需要一次性获取所有匹配结果时 |
| `iclude_reflect` | 多轮 LLM 推理 | 对记忆进行深度分析和推理 |
| `iclude_timeline` | 时间线查询 | 查看某段时间的记忆变化 |
| `iclude_ingest_conversation` | 批量导入对话 | 导入历史对话数据 |
| `iclude_create_session` | 创建会话上下文 | hook 自动调用，通常无需手动 |

**推荐工作流**: `iclude_scan` → 浏览索引 → `iclude_fetch` 获取详情

## 配置说明

配置文件位于 `~/.iclude/config.yaml`，以下是完整配置项及默认值：

```yaml
# ── 存储 ──
storage:
  sqlite:
    enabled: true
    path: "~/.iclude/iclude.db"       # 数据库文件路径
    tokenizer:
      provider: "simple"              # 分词器: simple | jieba | noop
  qdrant:
    enabled: false                    # 启用向量检索（需部署 Qdrant）
    url: "http://localhost:6333"
    collection: "memories"
    dimension: 1536

# ── LLM ──
llm:
  openai:
    api_key: "${OPENAI_API_KEY}"      # 支持环境变量引用
    model: "gpt-4o-mini"              # 用于摘要生成、实体抽取、反思推理
    base_url: ""                      # 自定义 API 地址（兼容 DeepSeek/Ollama 等）
  embedding:
    model: "text-embedding-3-small"
    dimension: 1536

# ── MCP Server ──
mcp:
  enabled: true
  port: 8081                          # HTTP+SSE 模式端口（stdio 模式不使用）
  default_team_id: "default"
  default_owner_id: "local-user"

# ── 自动化 Hooks ──
hooks:
  enabled: true
  capture:
    skip_tools:                       # 不捕获的工具（黑名单）
      - Glob
      - Grep
      - ToolSearch
      - TaskCreate
      - TaskUpdate
      - TaskList
      - TaskGet
      - TodoWrite
    max_input_chars: 1000             # 工具输入截断长度
    max_output_chars: 500             # 工具输出截断长度
  session_start:
    inject_limit: 20                  # 会话开始时注入的记忆数
  session_stop:
    summary_limit: 50                 # 会话结束时汇总的记忆数

# ── 检索 ──
retrieval:
  graph_enabled: true                 # 启用知识图谱检索
  graph_depth: 2                      # 图谱遍历深度
  fts_weight: 1.0                     # 全文检索 RRF 权重
  qdrant_weight: 1.0                  # 向量检索 RRF 权重
  graph_weight: 0.5                   # 图谱检索 RRF 权重

# ── 反思引擎 ──
reflect:
  max_rounds: 3                       # 最大推理轮数
  token_budget: 4000                  # 单次推理 token 上限
  round_timeout: 30s                  # 单轮超时
  auto_save: true                     # 自动保存推理结论

# ── 自主巡检 ──
heartbeat:
  enabled: true
  interval: 6h                        # 巡检间隔
  contradiction_enabled: true         # 矛盾检测
  decay_audit_min_age_days: 7         # 衰减审计最小记忆年龄

# ── 后台调度 ──
scheduler:
  enabled: true
  cleanup_interval: 1h                # 过期记忆清理间隔
  access_flush_interval: 30s          # 访问计数刷盘间隔
```

### 使用兼容 API 的模型

IClude 的 LLM 接口兼容所有 OpenAI API 格式的服务。只需修改 `base_url`：

```yaml
# DeepSeek
llm:
  openai:
    api_key: "sk-..."
    model: "deepseek-chat"
    base_url: "https://api.deepseek.com/v1"

# 本地 Ollama
llm:
  openai:
    api_key: "ollama"
    model: "qwen2.5:7b"
    base_url: "http://localhost:11434/v1"
```

## 升级

### 自动升级（重新运行安装脚本）

```bash
# macOS / Linux
curl -fsSL https://raw.githubusercontent.com/MemoryGet/LocalMem/main/install.sh | bash

# Windows
irm https://raw.githubusercontent.com/MemoryGet/LocalMem/main/install.ps1 | iex
```

安装脚本会：
- 自动下载最新版本二进制并覆盖旧版
- **不会覆盖**已有的 `config.yaml`（保留用户配置）
- **不会覆盖**已有的 `.mcp.json`（保留现有 MCP 配置）
- 数据库 `iclude.db` 不受影响，迁移自动执行

### 手动升级

1. 访问 [Releases 页面](https://github.com/MemoryGet/LocalMem/releases)
2. 下载对应平台的 `iclude-mcp` 和 `iclude-cli`
3. 替换 `~/.iclude/bin/` 下的二进制文件
4. 重启 Claude Code

## Docker 部署（完整服务端）

如需运行完整服务端（含 API Server + Qdrant 向量搜索 + 文档解析服务）：

```bash
git clone https://github.com/MemoryGet/LocalMem.git
cd LocalMem

# 设置 API Key
export OPENAI_API_KEY="sk-..."

# 启动全部服务
docker-compose -f deploy/docker-compose.yml up -d
```

服务端口：
- API Server: `http://localhost:8080`
- MCP Server (SSE): `http://localhost:8081`
- Qdrant: `http://localhost:6333`

## 卸载

### macOS / Linux

```bash
curl -fsSL https://raw.githubusercontent.com/MemoryGet/LocalMem/main/install.sh | bash -s -- --uninstall
```

或手动：

```bash
rm -rf ~/.iclude
# 从 ~/.bashrc 或 ~/.zshrc 中移除 IClude PATH 行
# 从 ~/.claude/.mcp.json 中移除 iclude 条目
```

### Windows

```powershell
Remove-Item -Recurse -Force "$env:USERPROFILE\.iclude"
# 从用户 PATH 环境变量中移除 .iclude\bin
# 从 ~/.claude/.mcp.json 中移除 iclude 条目
```

## 常见问题

### Claude Code 中看不到 IClude 工具

1. 确认 `~/.claude/.mcp.json` 中有 `iclude` 条目
2. 确认 `iclude-mcp` 在 PATH 中：`which iclude-mcp`（macOS/Linux）或 `where iclude-mcp`（Windows）
3. 重启 Claude Code
4. 在 Claude Code 中输入 `/mcp` 查看 MCP 服务器状态

### API Key 相关

- 安装时跳过了 API Key：设置环境变量 `export OPENAI_API_KEY="sk-..."`
- 更换 API Key：编辑 `~/.iclude/config.yaml` 中的 `llm.openai.api_key`
- 使用其他模型：修改 `llm.openai.base_url` 和 `model`（见上方配置说明）

### 数据存储

- 所有数据存储在本地 `~/.iclude/iclude.db`（SQLite）
- 数据库迁移在首次启动时自动执行，版本升级时自动增量迁移
- 备份：复制 `~/.iclude/iclude.db` 文件即可

### 中文搜索优化

默认分词器为 `simple`（按 CJK 字符逐字切分）。如需更精确的中文分词：

```yaml
storage:
  sqlite:
    tokenizer:
      provider: "jieba"
      jieba_url: "http://localhost:8866"
```

需同时启动 jieba 分词服务：`python tools/jieba_server.py`
