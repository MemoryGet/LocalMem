## 本轮关键修复清单（累计约 30 项）

### 数据库层
- Delete/SoftDelete/Restore/CleanupExpired/PurgeDeleted FTS5 操作全部事务原子化
- V9 迁移：实体名 COLLATE NOCASE 索引、uri/document_id/parent_id 索引、(team_id,visibility,owner_id) 复合索引
- IncrementMemoryCountBy 批量递增 + DecrementMemoryCount 下溢防护
- SoftDelete FTS5 错误记录 Warn 日志
- 迁移版本控制统一（migrateAddPerformanceIndexes 纳入 V9）

### AI/检索层
- Qdrant 空壳 Memory 回填（backfillMemories）+ 创建副本避免共享指针变异
- FindEntitiesByName 索引查询替代 500 硬限全量扫描
- Extractor.resolveEntity 两阶段（精确匹配 → LLM 规范化）
- RRF memMap 优先保留 Content 非空的 Memory
- 图谱遍历 sort.Slice 替代冒泡排序
- OpenAI embedder 指数退避重试（3次, 429/5xx）
- FTS5 MATCH 输入净化（sanitizeFTS5Query）

### 安全层
- MCP/REST 双端 Token 恒定时间比较（crypto/subtle）
- Restore/Reinforce OwnerID 权限检查
- Cleanup IsSystem() 管理员门控
- REST API 4MB Body 限制 + 分级速率限制（写 20rps, 搜索 10rps, LLM 2rps）
- MCP 速率限制（10rps/burst 20）
- MCP iclude_fetch 改用 GetVisible + IdentityFromContext
- HTTP 安全头（X-Content-Type-Options, X-Frame-Options, Referrer-Policy）
- RateLimitMiddleware 惰性清理替代 goroutine 泄漏

### 架构层
- config.GetConfig() 全局单例从业务层完全清除（5 处）
- ManagerConfig / ConsolidationConfig / HeartbeatConfig 构造函数注入
- Extractor 通过 GraphManager 公共方法访问（不再穿透内部字段）
- consolidation.go 编号 bug 修复
- IngestConversation 批量计数替代逐条循环
