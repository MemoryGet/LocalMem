# IClude 术语表 / Glossary

> 本文档列出 IClude 记忆体系统中所有领域名词及其含义。新增名词时请在对应分类下追加。
>
> 最后更新：2026-04-03

---

## 1. 核心概念

| 术语 | 英文 | 含义 |
|------|------|------|
| 记忆 | Memory | 系统中的基本数据单元，包含内容、元数据、嵌入向量、生命周期字段等共 37 列 |
| 记忆层级 | Memory Class | 记忆演化层级：`episodic`（原始事件）→ `semantic`（提炼规律）→ `procedural`（策略结论） |
| 保留等级 | Retention Tier | 记忆的持久化级别，决定衰减速度和过期策略 |
| 强度 | Strength | 记忆的当前有效强度（0~1），受衰减和强化影响 |
| 衰减率 | Decay Rate | 记忆强度随时间衰减的速率，不同保留等级有不同默认值 |
| 强化 | Reinforce | 对记忆施加正反馈，增加强度和 reinforced_count |
| 软删除 | Soft Delete | 标记 deleted_at 但不物理删除，可恢复 |
| 来源追踪 | Derived From | JSON 数组，记录一条记忆由哪些源记忆演化而来 |

---

## 2. 保留等级（Retention Tier）

| 值 | 含义 | 半衰期 | 过期策略 |
|----|------|--------|---------|
| `permanent` | 永久记忆 | 无衰减 | 不过期 |
| `long_term` | 长期记忆 | ~29 天 | 按衰减清理 |
| `standard` | 标准记忆（默认） | ~2.9 天 | 按衰减清理 |
| `short_term` | 短期记忆 | ~14 小时 | 按衰减清理 |
| `ephemeral` | 临时记忆 | ~7 小时 | 24 小时后过期 |

---

## 3. 记忆层级（Memory Class）

| 值 | 含义 | 来源 | 检索权重 |
|----|------|------|---------|
| `episodic` | 原始事件/事实记忆 | 外部写入（retain/ingest） | 1.0x |
| `semantic` | 观察/提炼/规律记忆 | consolidation 合并产出；heartbeat 晋升 | 1.2x |
| `procedural` | 策略/规则/心智模型 | reflect 推理产出 | 1.5x |

| `MemoryWriter` | 6 | 记忆写入操作（Create、Update、Delete、Reinforce 等） |
| `MemorySearch` | 2 | 全文检索（SearchText、SearchTextFiltered） |

| `ContextStore` | 10 | 层级上下文管理（树形结构，物化路径） |
| `TagStore` | 10 | 标签管理及记忆-标签关联 |
| `GraphStore` | 14 | 知识图谱：实体 + 关系 + 记忆-实体关联 |

## 8. 检索层

### 核心概念

| 术语 | 含义 |
|------|------|
| Retriever | 单轮检索引擎，协调三路检索通道 |
| Preprocessor | 查询预处理器：意图分类、同义词扩展、HyDE、时间窗口解析 |
| QueryPlan | 预处理输出的查询执行计划 |
| QueryIntent | 查询意图分类 |

### 查询意图（Query Intent）

| 值 | 含义 | 特征 |
|----|------|------|
| `IntentKeyword` | 精确查找 | 短查询、无探索性词汇 |
| `IntentSemantic` | 模糊/探索性查找 | 长查询或含 how/why/what |
| `IntentTemporal` | 时间相关查找 | 含时间关键词 |
| `IntentRelational` | 关联查找 | 含"相关""关于"等词 |
| `IntentGeneral` | 通用查找 | 默认 |

### 三路检索通道

| 通道 | 后端 | 说明 |
|------|------|------|
| FTS（全文检索） | SQLite FTS5 + BM25 | 关键词匹配，3 列加权（content 10, abstract 5, summary 3） |
| Semantic（语义检索） | Qdrant 向量搜索 | 余弦相似度 |
| Graph（图谱检索） | 知识图谱遍历 | 实体关系扩展 |

### 融合算法

| 算法 | 含义 |
|------|------|
| RRF（Reciprocal Rank Fusion） | 倒数排名融合，公式：`score = Σ weight × 1/(k + rank + 1)`，k=60 |
| MergeWeightedRRF | 带权重的 RRF 融合（fts_weight, qdrant_weight, graph_weight） |

### 重排序

| 算法 | Provider | 含义 |
|------|----------|------|
| OverlapReranker | `overlap` | 基于 token 重叠率的重排序 |
| RemoteReranker | `remote` | 远程 cross-encoder API 重排序 |
| MMR（Maximal Marginal Relevance） | — | 多样性重排序，lambda 控制相关性 vs 多样性权衡 |

### 权重体系

| 权重类型 | 对象 | 值 | 作用点 |
|---------|------|------|--------|
| kindWeights | skill | 1.5x | 类型加权 |
| kindWeights | profile | 1.2x | 类型加权 |
| kindWeights | fact/note | 1.0x | 类型加权 |
| subKindWeights | pattern/case | 1.3x | 子类型加权 |
| classWeights | procedural | 1.5x | 层级加权 |
| classWeights | semantic | 1.2x | 层级加权 |
| classWeights | episodic | 1.0x | 层级加权 |
| weightCap | — | 2.0 | 叠乘上限 |
| strengthWeight | — | 基于衰减计算 | 强度加权 |
| accessAlpha | — | 0.15 | 访问频率阻尼系数 |

### 时间窗口规则（Temporal Window Rules）

| 关键词 | 窗口大小 | 中心偏移 |
|--------|---------|---------|
| today / 今天 | 1 天 | 0 |
| yesterday / 昨天 | 1 天 | -1 天 |
| 前天 | 1 天 | -2 天 |
| this week / 本周 / 这几天 | 7 天 | 0 |
| last week / 上周 | 7 天 | -7 天 |
| this month / 本月 | 30 天 | 0 |
| last month / 上月 | 30 天 | -30 天 |
| last quarter / 上季度 | 90 天 | -90 天 |
| recent months / 最近几个月 | 90 天 | 0 |
| this year / 今年 | 365 天 | 0 |
| last year / 去年 | 365 天 | -365 天 |
| 默认（未匹配） | 30 天 | 0 |

## 9. 反思层（Reflect）

| 术语 | 含义 |
|------|------|
| ReflectEngine | 多轮 LLM 推理引擎 |
| ReflectRound | 单轮反思记录：查询 → 检索 → LLM 推理 → 决定是否继续 |
| ReflectResponse | 完整反思结果：结论 + 轮次追踪 + 元数据 |
| maxRounds | 最大推理轮数 |
| tokenBudget | Token 预算上限 |
| roundTimeout | 单轮超时时间 |
| autoSave | 自动保存结论为 procedural 记忆 |
| action | LLM 输出动作：`need_more`（继续检索）/ `conclusion`（输出结论） |
| adaptiveTopK | 自适应 Top-K：首轮 15、后续轮 8，按 token budget 和 round 动态调整 |
| priorRounds | 累积历史轮次的推理摘要，注入下一轮 context |
| queryDedup | 查询去重：防止跨轮重复查询导致无限循环 |
| parseFallback | 4 级解析降级：JSON → Regex → Retry → Raw Fallback |
| ContextID | 可选参数，指定 Context 以加载行为约束注入 prompt |
| Mission | Context 行为字段：上下文使命/目标，注入 Reflect 系统 prompt |
| Directives | Context 行为字段：行为指令（换行分隔），注入 Reflect 系统 prompt |
| Disposition | Context 行为字段：回复风格/性格，注入 Reflect 系统 prompt |

## 10. 合并层（Consolidation）

| 术语 | 含义 |
|------|------|
| Consolidator | 记忆合并引擎 |

## 11. 心跳巡检（Heartbeat）

| 术语 | 含义 |
|------|------|
| Engine | HEARTBEAT 自主巡检引擎，定期执行后台维护 |
| DecayAudit | 衰减审计：软删除强度过低的僵尸记忆 |
| OrphanCleanup | 孤儿清理：移除已软删除记忆的图谱关联 |
| ContradictionCheck | 矛盾检测：用 LLM 比较相似记忆是否矛盾 |
| AbstractBackfill | 摘要回填：为缺失 abstract 的记忆生成摘要 |
| Promotion | 晋升：`episodic` → `semantic`（reinforced_count ≥ 阈值） |
| PromotionThreshold | 晋升阈值（默认 5 次强化） |

## 24. 评测框架（Eval）

| 术语 | 含义 |
|------|------|
| Runner | 评测运行器：播种 → 检索 → 评分 → 聚合 |
| EvalDataset | 评测数据集（种子记忆 + 查询用例） |
| SeedMemory | 种子记忆（Content + Kind + MemoryClass） |
| EvalCase | 评测用例（Query + Expected + Category + Difficulty） |
| EvalReport | 评测报告（指标 + 分类 + 难度分组） |
| Baseline | 基线快照（JSON 文件，用于回归检测） |
| HitRate | 命中率：至少有一个结果匹配期望关键词的查询占比 |
| MRR | Mean Reciprocal Rank：平均倒数排名 |
| NDCG@k | Normalized Discounted Cumulative Gain |
| Recall@k | Top-k 召回率 |
| LongMemEval | 外部标准评测数据集（每题独立 DB） |
| EvolutionAwareRetrieval | 演化评测：A/B 对比 flat vs evolved memory_class 效果 |
